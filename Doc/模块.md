## 1. 传感器模块

### 1.1 DHT11 (温湿度传感器)

| 项目 | 说明 |
|------|------|
| **位置** | `components/managed_wrappers/dht/` |
| **依赖组件** | `esp-idf-lib/dht` |
| **接口类型** | 单总线 (GPIO) |
| **引脚** | GPIO4 |

**工作原理:**
- 单总线协议，主机发起通信
- 每次读取 40 位数据 (湿度整数 + 湿度小数 + 温度整数 + 温度小数 + 校验和)
- 建议读取间隔 ≥ 2 秒

**API 接口:**
```c
esp_err_t dht11_init(uint8_t gpio_num);
esp_err_t dht11_read(dht11_data_t *data);
void dht11_deinit(void);

typedef struct {
    float temperature;  // 温度 ℃
    float humidity;     // 湿度 %
    uint8_t valid;      // 有效标志
} dht11_data_t;
```

**调用示例:**
```c
dht11_data_t data;
if (dht11_read(&data) == ESP_OK && data.valid) {
    printf("温度: %.1f℃, 湿度: %.1f%%\n", data.temperature, data.humidity);
}
```

### 1.2 BH1750 (光照传感器)

| 项目 | 说明 |
|------|------|
| **位置** | `components/managed_wrappers/bh1750/` |
| **依赖组件** | `esp-idf-lib/bh1750`, `esp-idf-lib/i2cdev` |
| **接口类型** | I2C |
| **引脚** | SDA=GPIO15, SCL=GPIO16 |
| **I2C地址** | 0x23 |

**工作原理:**
- I2C 接口数字光强度传感器
- 工作模式：连续高分辨率模式 (分辨率 1 lux)
- 测量范围：1 - 65535 lux

**API 接口:**
```c
esp_err_t bh1750_sensor_init(uint8_t sda_gpio, uint8_t scl_gpio);
esp_err_t bh1750_sensor_read(float *lux);
void bh1750_sensor_deinit(void);
```

**调用示例:**
```c
float lux;
if (bh1750_sensor_read(&lux) == ESP_OK) {
    printf("光照强度: %.1f lux\n", lux);
}
```

### 1.3 MQ-2 (烟雾/可燃气体传感器)

| 项目 | 说明 |
|------|------|
| **位置** | `components/mq2/` |
| **接口类型** | ADC (模拟信号) |
| **引脚** | GPIO7 (ADC_CHANNEL_6) |
| **量程** | 0-4095 (12位 ADC) |

**工作原理:**
- 气敏电阻原理：可燃气体浓度越高，电阻越低，输出电压越高
- 使用 ESP-IDF ADC Oneshot API 读取
- 需要预热时间 (约 1-2 分钟)

**API 接口:**
```c
esp_err_t mq2_init(adc_channel_t channel);
esp_err_t mq2_read(adc_channel_t channel, uint32_t *value);
bool mq2_is_smoke_detected(uint32_t value, uint32_t threshold);
void mq2_deinit(void);
```

**阈值配置:**
```c
#define SMOKE_THRESHOLD 2500  // ADC 值超过此阈值触发报警
```

### 2.3 窗帘/舵机控制

| 项目 | 说明 |
|------|------|
| **位置** | `components/managed_wrappers/servo/` |
| **依赖组件** | `espressif/servo` |
| **技术** | PWM (50Hz) |
| **引脚** | GPIO8 |
| **脉宽范围** | 500μs (0°) - 2500μs (180°) |

**API 接口:**
```c
esp_err_t motor_init(uint8_t servo_gpio);
esp_err_t curtain_control(uint8_t open);  // 1=打开, 0=关闭
esp_err_t servo_set_angle(float angle);   // 0-180°
void motor_deinit(void);
```

**控制原理:**
```
PWM 周期: 20ms (50Hz)
0°   → 脉宽 0.5ms  → 占空比 2.5%
90°  → 脉宽 1.5ms  → 占空比 7.5%
180° → 脉宽 2.5ms  → 占空比 12.5%
```

### 3.3 语音识别核心

| 项目 | 说明 |
|------|------|
| **位置** | `components/sr/voice_recognition.c` |
| **依赖组件** | `espressif/esp-sr` |
| **唤醒词模型** | wn9_nihaoxiaozhi ("你好小智") |
| **命令词模型** | mn7_cn (中文) |

**双任务架构:**

| 任务 | CPU | 优先级 | 栈大小 | 职责 |
|------|-----|--------|--------|------|
| `vr_feed_task` | 0 | 5 | 4KB | I2S 读取 → AFE 输入 |
| `vr_detect_task` | 1 | 3 | 8KB | AFE 输出 → 模型检测 |

**状态机:**
```
┌─────────────────────────┐
│  WAITING_WAKE           │  ←─────────────────────────┐
│  等待唤醒词 "你好小智"   │                            │
└───────────┬─────────────┘                            │
            │ 唤醒                                      │
            ▼                                          │
┌─────────────────────────┐                            │
│  WAITING_COMMAND        │ ─────── 超时 (15s) ────────┤
│  等待命令词              │                            │
└───────────┬─────────────┘                            │
            │ 识别到命令                                │
            └── (保持等待，连续对话) ──────────────────┘
```

**支持的语音命令:**

| 命令词 | 拼音 | 功能 |
|--------|------|------|
| 打开灯光 | da kai deng guang | 开灯 |
| 关闭灯光 | guan bi deng guang | 关灯 |
| 打开风扇 | da kai feng shan | 开风扇 |
| 关闭风扇 | guan bi feng shan | 关风扇 |
| 打开窗帘 | da kai chuang lian | 开窗帘 |
| 关闭窗帘 | guan bi chuang lian | 关窗帘 |
| 红色 | hong se | RGB 变红 |
| 绿色 | lv se | RGB 变绿 |
| 蓝色 | lan se | RGB 变蓝 |
| 自动模式 | zi dong mo shi | 切换自动控制 |
| 手动模式 | shou dong mo shi | 切换手动控制 |

**API 接口:**
```c
esp_err_t vr_init(int sck_io, int ws_io, int sd_io, vr_command_callback_t callback);
esp_err_t vr_start(void);
esp_err_t vr_stop(void);
esp_err_t vr_deinit(void);
void vr_set_vad_callback(vr_vad_callback_t callback);
```

**回调机制:**
```c
// 命令回调
typedef void (*vr_command_callback_t)(vr_command_t command);

// VAD 状态回调
typedef void (*vr_vad_callback_t)(vr_vad_state_t state);
```

### 5.2 应用控制逻辑 (app_control)

| 项目 | 说明 |
|------|------|
| **位置** | `components/app_control/` |
| **作用** | 自动化控制策略 + 语音命令处理 |

**控制策略:**

**温度控制 (带滞回):**
```c
阈值: 30℃(低速) / 32℃(中速) / 35℃(全速)
滞回: 2℃

开启时: 温度 < (阈值 - 2℃) 才关闭
关闭时: 温度 > 阈值 才开启
```

**光照控制 (带滞回):**
```c
阈值: 100 lux
滞回: 20 lux

开启时: 光照 > 120 lux 才关灯
关闭时: 光照 < 100 lux 才开灯
```

**烟雾报警:**
```c
阈值: 2500 (ADC 值)
动作: 蜂鸣器报警 + 风扇全速通风
优先级: 任何模式下都生效
```

## 6. 模块集成原理

### 6.1 系统启动流程

```c
// main.c
void app_main(void) {
    app_start();  // 一键启动
}

// application.c - app_start_with_config()
1. app_state_init()           // 初始化共享状态
2. 创建信号量                  // 任务间同步
3. init_nvs()                 // NVS Flash
4. init_hardware()            // 所有硬件驱动
5. init_network()             // WiFi + HTTP
6. start_tasks()              // sensor_task + control_task
7. init_voice()               // 语音识别
```

### 6.2 任务调度

```
sensor_task (优先级 3)
    │
    ├── 读取 DHT11
    ├── 读取 BH1750
    ├── 读取 MQ2
    ├── 写入 app_state
    └── 通知 control_task (Semaphore)

control_task (优先级 4)
    │
    ├── 等待 sensor_task 通知
    ├── 读取 app_state
    ├── 执行控制逻辑
    └── 控制执行器 (LED/FAN/SERVO)

vr_feed_task (CPU0, 优先级 5)
    │
    ├── I2S 读取麦克风
    └── 喂入 AFE 处理器

vr_detect_task (CPU1, 优先级 3)
    │
    ├── AFE 获取处理后数据
    ├── WakeNet 唤醒检测
    ├── MultiNet 命令检测
    └── 回调 app_control
```

### 6.3 数据流向

```
传感器数据流:
硬件 → sensor_task → app_state → control_task/HTTP

控制命令流:
Web API/语音 → app_control → 硬件驱动
                    │
                    └→ app_state (同步状态)
```

### 6.4 模块依赖关系

```
config          ← 所有模块 (配置常量)
common          ← application, app_control, web_ui
managed_wrappers ← application, app_control
sr              ← application
web_ui          ← application
wifi            ← application
```

### 6.5 新增模块集成步骤

1. **创建驱动:** `components/新模块/`
2. **更新 CMakeLists.txt:** 添加源文件和依赖
3. **更新 app_types.h:** 添加新数据字段
4. **更新 application.c:** 在 init_hardware() 添加初始化
5. **更新 sensor_task/control_task:** 添加读取/控制逻辑
6. **更新 app_control.c:** 添加语音命令处理
7. **更新 http_server.c:** 添加 Web API
